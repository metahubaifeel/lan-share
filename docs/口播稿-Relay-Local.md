# Relay Local 开源介绍 · 口播稿

> 风格：口语化、分段短、自嘲、兄弟/对吧。约 4–6 分钟口播 + 可选 B-roll 提示。  
> 演示时建议：电脑开 Hub → 手机同 WiFi 打开 → 现场传图/传 zip/搜聊天记录。

---

## 开场（20 秒）

兄弟们，今天给你们看一个我自己用了十来天、准备开源的东西。

叫 **Relay Local**，局域网传输站。

就一句话：**同一 WiFi 下，任何设备打开一个网页，就能跟你的电脑互传文件、聊天、搜历史——不用微信文件传输助手，不用下 QQ，不用装 App。**

对吧，这事听起来应该很简单，但你自己传过 1.5G 的 zip 就知道，现有那些工具有多恶心。

---

## 痛点 / 为什么要做（90 秒）

我先说为什么做这玩意。

我日常有个特别真实的需求：手机拍的照片、录的视频、下的压缩包，要扔到电脑上；电脑上的东西，有时候也要发给手机看。

以前怎么走？**文件传输助手。** 对吧，大家都懂。

但文件传输助手有几个问题，我忍很久了：

第一，**它绑在微信里**。你不是在「传文件」，你是在「跟另一个聊天窗口说话」。体验很拧巴。

第二，**大文件、多文件很折磨**。压缩包传一半断了，图片被压画质，你根本不知道卡在哪。

第三，**你还得装一堆东西**。QQ、各种助手、各种「快传」——每个都要账号、都要生态。我就想要一个 **URL**，打开就能用，兄弟。

第四，我后来还有一个战略级的想法：文件传输助手很多人当 **私人记事本** 用——账号密码、API Key、重要链接，全扔里面，搜一下就能找到。

我也想有一个 **自己掌控** 的传输站，聊天记录在本地 SQLite，搜得到、删得掉、界面还好看。

所以我就想：**能不能在自己电脑上跑一个站，局域网内谁都能用？**

不要注册，不要下载，**同一 WiFi，输入 IP，完事。**

这就是 Relay Local。

---

## 它是什么（45 秒）

技术上极度克制：**一个 `server.py`，Python 标准库，零 pip 依赖。**

跑起来之后：

- 电脑浏览器：`http://localhost:8888`
- 手机浏览器：`http://你电脑局域网IP:8888`

顶栏一键复制地址，手机粘贴就能进。

它不是网盘产品，不是聊天 App 竞品——它是 **你家的传输枢纽**：手机往这儿扔，电脑收；电脑文件在这边浏览，手机也能下。

## 核心体验演示（2 分钟 · 建议边讲边录屏）

### 1. 聊天传文件（像文件传输助手，但更好）

打开 **聊天**  tab。

- 点 📎 或相册，**一次多选** 很多张 JPG、MP4、zip
- 上方出现 **待发文件栏**：缩略图、能预览、能单独删、能清空
- 多个文件会问你：**逐条发，还是打包成一个 zip？**——你自己选，不替你瞎打包
- 上传有 **全宽进度条**：百分比、速度、失败原因用中文告诉你，不是 silent fail

发出去之后，**电脑和手机两边聊天都能立刻看到**——跟微信发消息一样，不是「只有收件箱有、聊天里没有」。

### 2. 翻历史不被拽下去

这个我专门修过。以前每两秒轮询就把你拽到最底部，看历史简直反人类。

现在：**你往上翻，就不动你；你在底部，新消息才自动跟。**

### 3. 文件浏览 + 预览

**文件** tab：能逛你电脑上的目录（默认 `~/`，可配置）。

- 分类：图片 / 视频 / 音频 / 文档
- 排序：名称 / 大小 / 日期
- 电脑端左侧列表、右侧预览；手机点开会全屏预览
- 支持 Markdown、PDF、Word、HEIC……不是只能下不能看

手机看图库还有一个我很爱的设计：**按年份横滑的时间线**，像翻相册，不是一坨列表。

### 4. 搜索

右上角放大镜，或者 `Ctrl+K`。

搜聊天记录里的「账号」「密码」「某项目名」——**点结果直接跳到那条消息并高亮**。

本地模式还能搜文件名，逛整个 home 目录（自动跳过 `.git` 那种目录）。

### 5. 多选批量

手机 **长按** 进入多选，底部浮条：选了几张、批量下载、删除。

选中项上面还有 **横向预览条**，看得见你选了啥——不是盲选。

## 设计（45 秒 · 我最想吹的部分）

兄弟们，我知道你们很多人跟我一样——**功能能用不够，界面得好看。**

这个前端我磨了很久：

- **复古终端美学**：黑底、金色网格、JetBrains Mono，像一台老机器里的传输控制台
- 每个按钮都有 **按压反馈** 和轻微动效，手机上有 tap highlight，不塑料
- 可选 **8-bit 音效**——发消息「哔哔」一下，很上头；不喜欢一键关
- 压缩包在聊天里是 **卡片 + 归档图标**，不是一行丑文字
- 上传 / 下载都有 **banner 进度**，大文件下载会交给系统下载管理器，手机也不会假死

我不是要做「又一个文件管理器 UI」。我要的是 **有性格、有层级、用起来顺** 的传输站。

## 技术亮点（30 秒 · 给懂行的兄弟）

- 单文件部署，**无 npm、无 Docker 也行**
- 流式上传大文件，multipart 边界解析修过（手机传 GB 级 zip 不再截断）
- 下载支持 Range；缩略图缓存；SQLite 持久化聊天
- 路径穿越防护；dot 目录跳过
- systemd 用户服务模板，开机自启

开源的是 **局域网版**。我自己还有一个云上的私人实例 daily drive——那个配置不进仓库，你们本地 fork 自己玩就行。

---

## 谁适合用（30 秒）

- 家里 / 工作室 **多设备同一 WiFi**
- 不想把文件过一遍微信、不想被压缩
- 想要一个 **自己的、可搜索的** 传输 + 备忘站
- 能接受：**只在可信局域网用**（默认无登录，别裸奔到公网）

---

## 收尾（25 秒）

所以总结一下：

**Relay Local = 你自己电脑上的传输站。**

一个网页，所有设备都能进；传文件、聊天、搜索、预览、批量操作；界面是我认真做过设计的，不是后台管理风。

微信文件传输助手可以休息了吗？我不敢说完全替代——但 **至少在这套 WiFi 里，你不用再受那份气了。**
